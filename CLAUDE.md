## 🎯 Долгосрочный план: миграция со старого стэка на новый

Сейчас на проекте **два стэка параллельно**:
- **`(webflow)` group** — 537 страниц через `dangerouslySetInnerHTML` + Webflow CSS + jQuery/Slick/GSAP. Это legacy 1:1 клон оригинального sosmovingla.net
- **`(new-design)` group** — новый стэк: Next + React 19 + Tailwind 4 + framer-motion + Three.js + Lenis. Пока только `/mainpage2`

### Цель (когда начнётся редизайн)
Постепенно **мигрировать весь сайт на новый стэк** (Tailwind+React+motion), компонент за компонентом, страница за страницей:

1. **Мастер-компоненты на новом стэке** — Navbar, Footer, Hero, CTA, Reviews, FAQ, Services и т.д. (уже частично есть в `(new-design)`)
2. **Шаблоны страниц** — `/[citySlug]` и `/services/[slug]` переделать на React-компоненты, данные городов/услуг — в JSON
3. **Блог** — посты в Markdown + React Article компонент
4. **После миграции** — Webflow HTML, jQuery, webflow.css, ScriptLoader **удаляются**. Остаётся один современный стэк.

### Порядок миграции (рекомендованный)
1. Закончить Фазу 2 Webflow-копии (master-компоненты поверх текущего HTML — это черновик)
2. Начать переносить по одной странице: сначала простые (about, free-estimate), потом cities/services, последним — блог
3. Делать по ветке per миграция, проверять 1:1 визуал с прошлой версией
4. Когда все страницы переведены → удалить `(webflow)/` group и связанные ассеты
